# HR 工作流程板块换序设计

## 目标

将“HR工作流程建设”板块放到“策划”板块之后，并让菜单中的入口顺序与页面顺序保持一致。

## 实现范围

- 在 `index.html` 中移动完整的 `#hr-workflow` 区块，使其紧跟在 `#planning` 区块之后。
- 在菜单中移动“HR工作流程建设”入口，使其位于“策划”入口之后。
- 保留现有区块 ID、链接地址、文案、样式类和交互逻辑。

## 不变项

- 不修改板块内部内容。
- 不修改其他板块的相对顺序。
- 不使用 CSS `order` 或 JavaScript 动态重排，确保 DOM 顺序、视觉顺序和键盘导航顺序一致。

## 验收标准

- 页面中“策划”后紧接“HR工作流程建设”。
- 菜单中“策划”后紧接“HR工作流程建设”。
- 菜单锚点仍能跳转到对应板块。
- 项目构建成功。
